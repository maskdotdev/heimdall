# Schema Examples

This directory contains compact examples that show how schema payloads fit together.

The examples are review aids, not generated fixtures. Once services and workers exist, move executable fixtures under `tests/fixtures` and validate them with the runtime validators for those services.

## Files

- `review-flow.example.json`: A minimal end-to-end shape showing repository metadata, a review run, raw LLM output, a validated finding, and a publishable review.
